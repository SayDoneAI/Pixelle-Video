# Template Studio Editor Spec

## 当前实现范围（Editor Core V1）

本轮先落“编辑器内核”，不继续扩张参数表单：

- 保持 `Template Studio` 为独立页面
- 固定支持 `1080x1440/image_sketch_card.html`
- 用显式元素模型映射当前模板
- 中间区域改为可点选画布
- 右侧仅显示当前选中元素属性
- `text` 明确作为动态字幕槽位单独处理
- 继续复用现有 HTML 预览链路

暂不包含：

- 空白添加元素
- 保存 / 重载模板结构
- 自由拖拽与缩放
- 多模板支持

## 目标

把当前 `Template Studio` 从“参数面板页”升级为“直接操作型模板编辑器”。

这个编辑器不是通用海报工具，也不是普通的静态画布编辑器，而是：

- 面向短视频分镜模板
- 同时支持静态元素和动态内容槽位
- 重点服务 `title` / `text` / `image` 这类会随分镜变化的数据字段

一句话定义：

`Template Studio` 应该是一个“带动态字段槽位的模板编辑器”。

## 当前问题

当前 V1 已经解决了“独立页面 + 可调参数 + 可刷新预览”，但体验仍然偏工程化：

- 用户在填参数表，不是在编辑画布
- 参数名暴露为 `title_bar_height`、`caption_padding_x` 这类工程字段
- 视觉上像后台配置，不像创作工具
- 用户无法通过“点选画布元素”建立心智映射
- “字幕”被当成普通文本对待，但它实际上是每一帧都会变化的动态内容

## 设计原则

### 1. 画布优先

模板创作必须是以画布为中心，而不是以表单为中心。

- 中间是主画布
- 右侧才是属性面板
- 左侧只承载模板/图层/数据预设

### 2. 直接操作优先

用户应尽量通过点击画布完成编辑：

- 点元素 = 选中元素
- 点文本 = 进入文本编辑或属性编辑
- 点图片 = 替换图片或调图片规则
- 点空白 = 添加元素入口

### 3. 动态字段和静态元素严格区分

模板里至少有三类元素：

- `StaticText`
  固定不变的文本，例如品牌名、免责声明、固定标签
- `DynamicField`
  绑定到分镜数据的内容槽位，例如 `title`、`text`、`image`
- `Decoration`
  装饰层，例如背景块、分隔线、边框、角标、蒙层

### 4. 字幕不是普通文本

字幕属于 `DynamicField(text)`。

编辑字幕块时，编辑器不应该默认让用户改“某一句话”，而应该让用户改：

- 位置
- 容器尺寸
- 字号
- 行高
- 字重
- 对齐方式
- 内边距
- 最大行数
- 溢出策略
- 是否自动缩小

也就是说，字幕编辑的是“排版规则”，不是某一帧的实际内容。

### 5. 预览必须依赖样例数据

动态字段必须有样例数据才能预览。

至少提供三组样例：

- `短字幕`
- `中字幕`
- `长字幕`

这样用户能在模板编辑阶段直接看到：

- 短文案时画面是否太空
- 长文案时是否溢出
- 字幕块是否需要自动缩小或裁切

## 推荐页面结构

## 顶部工具栏

- 模板名称
- 当前模板尺寸
- 当前预览数据集
- 撤销 / 重做
- 保存
- 刷新预览

不放：

- LLM 配置
- TTS 配置
- ComfyUI 配置
- 视频生成按钮

## 左侧栏

优先承载两类内容：

- `图层列表`
  显示当前模板中的元素层级
- `预览数据`
  切换短字幕 / 中字幕 / 长字幕 / 测试图片

V2 可以先只做图层列表和预览数据，不强求素材库。

## 中间主画布

主画布是整个产品的核心。

建议形态：

- 固定手机画布容器
- 白色或中性灰工作台背景
- 被选中元素显示描边、控制点或标签
- 动态字段显示槽位标记，如：
  - `[标题]`
  - `[字幕]`
  - `[插图]`

这样用户能清楚知道哪些内容是模板固定的，哪些会被运行时替换。

## 右侧属性面板

右侧只展示“当前选中元素”的属性，不展示全量参数表。

### 选中 `StaticText`

展示：

- 文本内容
- 字号
- 字重
- 行高
- 对齐
- 颜色
- 位置与尺寸

### 选中 `DynamicField(title)`

展示：

- 字段类型：标题
- 字号
- 字重
- 对齐
- 容器尺寸
- 行数限制
- 溢出策略

### 选中 `DynamicField(text)`

展示：

- 字段类型：字幕
- 容器宽高
- 字号
- 行高
- 段落间距
- 最大行数
- 自动缩小开关
- 截断/省略/溢出策略
- 垂直对齐方式

### 选中 `DynamicField(image)`

展示：

- 字段类型：插图
- 位置与尺寸
- 填充方式：`contain` / `cover`
- 裁切方式
- 圆角
- 边框
- 留白

### 选中 `Decoration`

展示：

- 颜色
- 透明度
- 尺寸
- 圆角
- 阴影
- 层级

## 关键交互规则

### 点击行为

- 单击元素：选中
- 双击静态文本：直接编辑文本
- 单击动态字段：进入属性编辑，不直接把样例值当模板内容保存
- 单击空白：弹出添加菜单

### 空白区域添加

空白点击后应出现 `+ 添加元素` 菜单。

菜单项分三组：

- 静态内容
  - 静态标题
  - 静态正文
  - 静态标签
- 动态字段
  - 动态标题
  - 动态字幕
  - 动态图片
- 装饰元素
  - 分隔线
  - 背景块
  - 边框

注意：

- “动态字幕”不能伪装成普通文本框
- 动态字段必须有明确字段类型

### 选中状态

被选中元素必须有明确视觉反馈：

- 2px 描边
- 浮动标签
- 对应图层高亮

不要只靠右侧面板变化来暗示选中状态。

### 拖拽与缩放

这一层建议延后做，不要和点选编辑一起首发。

优先顺序应该是：

1. 先点选
2. 再属性编辑
3. 再做拖拽和缩放

否则交互成本会陡增。

## 字幕专属规则

这是整个编辑器和普通海报工具最不同的地方。

### 字幕块的本质

字幕块不是“固定文本”，而是：

- 一个动态字段容器
- 承载多条不同长度文本
- 需要对极端情况保持稳定

### 字幕块需要重点验证三类场景

- 短字幕：一两行
- 中字幕：三到五行
- 长字幕：接近上限

### 字幕块建议支持的排版策略

- 自动缩小字号
- 限制最大行数
- 溢出裁切
- 底部对齐 / 顶部对齐
- 固定容器高度

V2 至少要做到：

- 可切换预览数据集
- 可在字幕属性面板中调整字号、行高、最大行数、容器尺寸

## 分阶段落地建议

## V2：从参数台升级为直接操作编辑器

目标：

- 画布可点选
- 右侧只显示当前元素属性
- 字幕作为 `DynamicField(text)` 特殊处理
- 提供短/中/长字幕样例切换

范围：

- 不做自由拖拽
- 不做任意新增元素
- 先只支持已有元素的点选编辑

这是最小但最有感知提升的一步。

## V2.5：空白添加静态元素和装饰元素

目标：

- 点空白可添加静态文本
- 点空白可添加分隔线 / 背景块等装饰元素

范围：

- 动态字段新增仍通过明确菜单操作
- 不开放完全自由布局系统

## V3：完整元素系统

目标：

- 统一元素模型
- 支持拖拽、缩放、删除、层级调整
- 支持新增动态标题 / 动态字幕 / 动态图片槽位

到这一层，产品才真正接近“可视化模板编辑器”。

## 数据模型建议

后续如果要支撑“点空白新增元素”，建议逐步过渡到显式元素模型。

示意：

```json
{
  "elements": [
    {
      "id": "title-field",
      "type": "dynamic-field",
      "field": "title",
      "x": 80,
      "y": 36,
      "w": 920,
      "h": 72,
      "style": {}
    },
    {
      "id": "subtitle-field",
      "type": "dynamic-field",
      "field": "text",
      "x": 68,
      "y": 1040,
      "w": 944,
      "h": 260,
      "style": {}
    },
    {
      "id": "brand-static",
      "type": "static-text",
      "text": "Pixelle-Video",
      "x": 820,
      "y": 1320,
      "w": 180,
      "h": 32,
      "style": {}
    }
  ]
}
```

短期内可以继续从现有 HTML 模板参数映射到这个模型。

长期则应让编辑器直接操作元素模型，再导出为模板。

## 验收标准

如果进入 V2 实作，至少满足以下标准：

- 用户能在画布中点选标题、字幕、图片、静态文本、装饰线
- 选中不同元素后，右侧面板内容会切换
- 字幕块有独立的动态字段编辑逻辑
- 可切换短/中/长字幕样例数据进行预览
- 用户不需要理解 `title_bar_height` 这类工程字段
- 页面视觉上更像编辑器，而不是后台配置页

## 不该做的事

- 不要继续把更多参数平铺成更长的表单
- 不要把字幕直接当成普通文本框
- 不要在第一版点选编辑里同时塞进拖拽、缩放、图层排序、吸附对齐
- 不要把视频生成、TTS、ComfyUI 等生产配置重新混回编辑页

## 结论

下一步最值得做的不是“把表单做漂亮”，而是：

1. 先把当前页升级成真正的点选式编辑器
2. 把字幕当成动态字段而不是普通文本
3. 让空白添加元素成为第二阶段，而不是第一步就做成大而全

这样做最省工程成本，也最符合用户的直觉。
